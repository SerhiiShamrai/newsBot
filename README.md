# 📰 Telegram-бот для моніторингу новин про завершення війни

Цей бот автоматично перевіряє новинні джерела двічі на день і публікує в Telegram-групу ті новини, що стосуються теми "завершення війни Росії проти України" (перемовини, перемир'я, мирна угода, припинення вогню, мирний план, капітуляція тощо).

## 🌟 Особливості

- **Семантичний пошук** — аналізує новини не за ключовими словами, а за змістом
- **Многоязкова модель** — використовує `paraphrase-multilingual-MiniLM-L12-v2` (підтримує українську)
- **Автоматичний розклад** — через GitHub Actions (6:00 та 18:00 за Києвом)
- **Без дублів** — зберігає історію опублікованих новин

## 📁 Структура проєкту

```
├── main.py                 # Точка входу
├── news_checker.py         # Логіка отримання та фільтрації новин
├── config.py               # Конфігурація (еталонні речення, RSS-джерела)
├── published.json          # Історія опублікованих новин
├── requirements.txt        # Залежності Python
└── .github/workflows/news.yml  # GitHub Actions workflow
```

## 🚀 Швидкий старт

### 1. Клонуйте репозиторій

```bash
git clone <YOUR_REPO_URL>
cd Alfred
```

### 2. Встановіть Python залежності

```bash
pip install -r requirements.txt
```

### 3. Отримайте Telegram токен та ID групи

#### Токен бота
1. Відкрийте [https://t.me/botfather](https://t.me/botfather) в Telegram
2. Почніть діалог з `@BotFather`
3. Створіть нового бота за командою `/newbot`
4. Отримайте токен (наприклад: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

#### ID групи/чат
1. Відкрийте [https://t.me/](https://t.me/) у браузері або будь-якому клієнті
2. Зайдіть у вашу групу/канал
3. Натисніть на імя групи → **Info** → **Edit** (або "Редагувати")
4. У полі **Username** скопіюйте ID (наприклад: `-100123456789` або `123456789`)

> 💡 Для приватних чатів/груп бот має бути доданий як учасник.

### 4. Додайте секрети в GitHub

1. Зайдіть у **Settings** → **Secrets and variables** → **Actions**
2. Створіть наступні секрети:

| Ім'я секрету | Приклад значення |
|-------------|------------------|
| `TELEGRAM_BOT_TOKEN` | `123456789:ABCdefGHIjklMNOpqrsTUVwxyz` |
| `TELEGRAM_CHAT_ID` | `-100123456789` |
| `GROQ_API_KEY` | `gsk_...` (ключ від Groq) |

#### Ключ Groq API

Щоб отримати ключ для AI-сумаризації новин:

1. Зайдіть на [https://console.groq.com](https://console.groq.com)
2. Створіть акаунт або увійдіть
3. Перейдіть у **API Keys** → **Create API Key**
4. Скопіюйте ключ (починається з `gsk_`)
5. Додайте його як секрет `GROQ_API_KEY` у GitHub

### 5. Перший запуск (завантаження моделі)

При першому запуску скрипт автоматично завантажить модель embeddings (~100 MB). Це може зайняти кілька хвилин.

### 6. Перевірте роботу в Actions

1. Зайдіть у **Actions** у репозиторії GitHub
2. Знайдіть workflow `news`
3. Зачекайте до розкладу (6:00 або 18:00 за Києвом)
4. Перевірте логи останнього запуску

## ⚙️ Налаштування

### Зміна еталонних речень та порогу схожості

Відкрийте файл `config.py` і знайдіть наступні розділи:

#### Еталонні речення (рядки 5-12)

```python
ETALON_SENTENCES = [
    "Україна і Росія підписали мирну угоду",
    "оголошено припинення вогню між Україною і Росією",
    "завершилась війна в Україні",
    "Росія погодилась на капітуляцію українських військ",
    "підписано мирний план врегулювання конфлікту",
    "припинення бойових дій між Україною та Росією",
    "Україна і Росія досягли домовленості про перемир'я"
]
```

> 💡 Додавайте або змінюйте речення за потреби. Модель автоматично підлаштовується під нові формулювання.

#### Порог схожості (рядок 14)

```python
SIMILARITY_THRESHOLD = 0.5  # від 0.0 до 1.0
```

- `0.3` — чутливіший пошук (більше новин, але може бути більше "шуму")
- `0.7` — суворіший пошук (менше новин, але точніші)
- `0.5` — баланс між кількістю та якістю

#### RSS-джерела (рядки 16-23)

```python
RSS_SOURCES = [
    {"name": "Українська правда", "url": "https://www.pravda.com.ua/rss/"},
    {"name": "BBC Ukrainian", "url": "https://www.bbc.com/ukrainian/rss"},
    {"name": "Reuters World News", "url": "https://www.reuters.com/world/europe/rss/"},
    {"name": "Суспільне Новини", "url": "https://suspilne.media/feed/news/"},
]
```

> 💡 Змінюйте посилання за потреби. Переконайтеся, що це робочі RSS-стрічки.

## 📊 Як працює семантичний пошук

1. **Завантаження моделі** — при першому запуску завантажується модель `paraphrase-multilingual-MiniLM-L12-v2` (~100 MB)
2. **Обчислення embeddings** — для кожного еталонного речення обчислюється векторне представлення
3. **Читання RSS** — завантажується контент з усіх RSS-джерел
4. **Фільтрація** — для кожної новини обчислюється схожість з еталонами
5. **Вибір релевантних** — новини зі схожістю > порогу додаються до списку
6. **Публікація** — релевантні новини публікуються в Telegram

## 🔧 Робота без GitHub Actions

Якщо ви хочете запускати скрипт вручну:

```bash
python main.py
```

Скрипт:
- Перевіряє RSS-джерела
- Фільтрує новини за семантикою
- Публікує в Telegram
- Зберігає опубліковані URL в `published.json`
